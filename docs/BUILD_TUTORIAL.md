# 🚀 Build the Multi-modal RAG Document Analyst — Step-by-Step

> A complete guide to building this project from scratch: environment setup, code, models, and running the app.

---

## What You'll Build

A fully local AI-powered document analyst that:
- Reads and understands PDFs (text **and** images/charts)
- Lets you ask natural language questions
- Runs 100% offline — no API keys needed

**Tech stack:** Python · Streamlit · Ollama · Llama 3 · Moondream · ChromaDB · PyMuPDF

---

## Phase 1 — Project Setup

### 1.1 Clone the repository

```bash
git clone https://github.com/your-username/rag-document-analyst.git
cd rag-document-analyst
```

> Or if starting fresh from the files directly:
> ```bash
> mkdir rag-document-analyst
> cd rag-document-analyst
> git init
> ```

### 1.2 Create a virtual environment

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

You should now see `(venv)` at the start of your terminal prompt.

### 1.3 Install Python dependencies

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

This installs:

| Package | Purpose |
|---|---|
| `streamlit` | Web UI framework |
| `ollama` | Python client for local LLMs |
| `chromadb` | Vector database |
| `sentence-transformers` | Text embedding model |
| `pymupdf` | PDF parsing & page rendering |
| `torch` + `torchvision` | GPU acceleration (optional) |
| `pillow` | Image handling |

> ⏳ First install takes 5–10 minutes due to PyTorch download size.

## Phase 2 — Download AI Models via Ollama

Ollama manages local LLM downloads. Make sure the Ollama app is **running** before these steps.

### 2.1 Start Ollama (if not already running)

```bash
# Windows / macOS: open the Ollama app from your Applications folder
# Linux:
ollama serve
```

### 2.2 Pull the text LLM (Llama 3)

```bash
ollama pull llama3
```

> ⏳ ~4.7 GB download. Takes 5–15 minutes depending on your connection.

### 2.3 Pull the vision model (Moondream)

```bash
ollama pull moondream
```

> ⏳ ~1.7 GB download.

### 2.4 Verify both models are available

```bash
ollama list
```

Expected output:
```
NAME            ID              SIZE    MODIFIED
llama3:latest   365c0bd3c000    4.7 GB  ...
moondream:latest f96c1d9e9a5c   1.7 GB  ...
```

## Phase 3 — Understand the Project Files

Before running, here's what each file does:

```
rag-document-analyst/
│
├── resources.py        ← Shared config + lazy singletons
│                          (loads embedding model & ChromaDB once)
│
├── ingest.py           ← PDF ingestion pipeline
│                          text → chunks → embeddings → ChromaDB
│                          images → Moondream description → ChromaDB
│
├── query_engine.py     ← RAG retrieval + answer generation
│                          query → embed → ChromaDB search → Llama 3
│
├── app.py              ← Streamlit UI
│                          sidebar, chat, upload, document library
│
└── requirements.txt    ← Python dependencies
```

## Phase 6 — Using the App

### Upload your first document

1. In the **left sidebar**, click **"Browse files"** under *Upload Document*
2. Select any PDF from your machine
3. Click the **"⚡ Process & Add to Library"** button
4. Watch the progress bar — VLM page description takes ~10–30 s per page
5. When it completes, the doc appears in the **Document Library**

### Ask a question

1. Type in the **chat input** at the bottom
2. Press **Enter**
3. The AI retrieves relevant chunks and generates an answer
4. Source images are shown inline when visual content is cited

### Scope your search

- Use the **Query Scope** dropdown to ask about **one specific document** or **all documents**
- A blue banner appears when you are in scoped mode

### Manage documents

```
🗑️ (icon next to a doc)   → Delete that document's chunks
🗑️ Clear Entire Library   → Wipe all chunks from the DB
```

## Phase 7 — Common Configuration Changes

All constants are in `resources.py`:

```python
PERSIST_PATH    = "./chroma_db"        # where ChromaDB stores data
COLLECTION_NAME = "multimodal_docs"    # collection name
EMBED_MODEL_ID  = "all-MiniLM-L6-v2"  # embedding model
TEXT_LLM        = "llama3"            # text answer model
VISION_LLM      = "moondream"         # page image description model
```

Chunking settings in `ingest.py`:

```python
CHUNK_SIZE    = 500   # characters per chunk (increase for more context per result)
CHUNK_OVERLAP = 50    # overlap between chunks (increase to reduce boundary loss)
```

Retrieval setting in `query_engine.py`:

```python
TOP_K = 6   # how many chunks to retrieve per query (increase for richer context)
```

## Troubleshooting

| Problem | Fix |
|---|---|
| `Connection refused` on port 8501 | Firewall blocked it — try `streamlit run app.py --server.port 8502` |
| `ollama: connect ECONNREFUSED` | Ollama is not running — start the Ollama app |
| `model not found: llama3` | Run `ollama pull llama3` |
| `CUDA out of memory` on first query | Wait 15 seconds after ingestion, then retry |
| Very slow ingestion | Normal — Moondream runs page-by-page. A 10-page PDF takes ~3 min |
| `ModuleNotFoundError` | Make sure `(venv)` is active and `pip install -r requirements.txt` was run |
| ChromaDB errors after crashing | Delete `chroma_db/` folder and re-ingest your PDFs |

### Reset everything

```bash
# Delete the vector store (forces re-ingestion next time)
rm -rf chroma_db/         # macOS/Linux
Remove-Item -Recurse chroma_db\   # Windows PowerShell

# Clear temp processing files
rm -rf temp_uploads/ temp_images/     # macOS/Linux
Remove-Item -Recurse temp_uploads\, temp_images\  # Windows PowerShell
```
